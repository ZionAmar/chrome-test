# מה עכשיו — רשימה ברורה

## אם עדיין לא העלית ל-GitHub

1. פתח PowerShell בתיקיית הפרויקט והרץ:
   ```
   git remote add origin https://github.com/ZionAmar/chrome-test.git
   git branch -M main
   git push -u origin main
   ```
2. ב-GitHub: **Settings** → **Pages** → Source: **main** / **/ (root)** → Save.
3. חכה דקה־שתיים. קישור מדיניות פרטיות:
   **https://zionamar.github.io/chrome-test/landing/privacy.html**

---

## אם חנות Chrome עדיין לא נשלחה לאישור

1. **דף רישום (Store listing):** תיאור מפורט, אייקון 128×128, לפחות צילום מסך אחד, קטגוריה (Tools), דף הבית, תמיכה, תוכן למבוגרים = לא.
2. **Privacy:** כל ההצדקות (Single purpose, storage, activeTab, Host), Remote code = No, סמן Website content + שלוש התיבות, **Privacy policy URL** =  
   `https://zionamar.github.io/chrome-test/landing/privacy.html`
3. **שמור טיוטה** → **Submit for review**.

---

## אחרי ששלחת לאישור

- מחכים למייל מגוגל (בדרך כלל כמה ימים).
- אם יבקשו תיקונים — תקן לפי ההערות ושלח גרסה חדשה.
- אם אושר — התוסף יופיע בחנות והקישור יהיה בממשק המפתחים.

---

## אחרי שהתוסף פורסם (להכנסות)

1. **LemonSqueezy:** צור מוצר 5$ חד־פעמי, הפעל License keys, העתק Checkout URL ו-API key.
2. **Vercel:** העלה את **api/** עם **vercel.json**, הוסף LEMON_SQUEEZY_API_KEY, קבל כתובת כמו  
   `https://xxx.vercel.app/api/verify`
3. **config.js:** שים CHECKOUT_URL ו-VERIFY_API_URL האמיתיים, הרץ **build.ps1**, העלה גרסה חדשה של התוסף לחנות.

מכאן והלאה — משתמשים מורידים מהחנות, חודש חינם, ואז 5$ בלחיצה והכסף אליך.

# סגירת כל הנקודות לפני פרסום

רשימת הדברים שהחנות ביקשה — ואיפה לעשות כל אחד.

---

## 1. נוהלי פרטיות (Privacy practices) — כרטיסייה נפרדת

בדף עריכת הפריט יש כרטיסייה **"Privacy practices"** / **"נוהלי פרטיות"**. שם יש למלא:

| מה החנות מבקשת | איפה למלא | איפה מוכן להעתקה |
|-----------------|------------|-------------------|
| הצדקה ל-activeTab | שדה ההצדקה של activeTab | **PRIVACY_PRACTICES_STORE.md** סעיף 2 |
| הצדקה להרשאת מארח (host) | שדה ההצדקה של host permission | **PRIVACY_PRACTICES_STORE.md** סעיף 3 |
| הצדקה לקוד מרחוק | שדה ההצדקה של remote code | **PRIVACY_PRACTICES_STORE.md** סעיף 5 |
| הצדקה ל-storage | שדה ההצדקה של storage | **PRIVACY_PRACTICES_STORE.md** סעיף 4 |
| תיאור המטרה היחידה (Single purpose) | שדה Single purpose | **PRIVACY_PRACTICES_STORE.md** סעיף 1 |
| אישור תאימות למדיניות השימוש בנתונים | תיבת סימון | סמן שהשימוש בנתונים תואם את מדיניות התוכנית |

פתח את **PRIVACY_PRACTICES_STORE.md** — כל הטקסטים שם באנגלית, מוכנים להעתקה־הדבקה לשדות המתאימים.

---

## 2. צילום מסך או סרטון

- **מה לעשות:** העלה **לפחות תמונה אחת** ב־Screenshots (ב־Localized או Global).
- **גודל:** 1280×800 או 640×400, JPEG או PNG (ללא alpha).
- **איך:** פתח **store-assets/screenshot-promo.html** בדפדפן, צלם מסך, שמור כ־JPEG והעלה.  
  (פירוט ב־**שדות_חנות_Chrome.md**.)

## 3. תמונת סמל (Store icon)

- **מה לעשות:** בכרטיסיית Graphic assets / Store icon — גרור או העלה את **icons/icon128.png** (128×128) מהפרויקט.
- **מיקום בפרויקט:** `icons/icon128.png`

## 4. קטגוריה

## 5. תיאור מפורט (Detailed description)

- **מה לעשות:** וודא שבשדה "Detailed description" יש **לפחות 25 תווים** (החנות לפעמים כותבת 25 אבל מתכוונת ליותר — עדכן לתיאור מלא).

## 6. כתובת דוא"ל ליצירת קשר + אימות

- **איפה:** כרטיסייה **"Account"** / **"חשבון"** (בדשבורד המפתחים, לא בתוך הפריט).
- **מה לעשות:**  
  1. הזן **Contact email** — כתובת המייל שלך ליצירת קשר.  
  2. לחץ על **"Verify"** / **"אימות"** (או "Send verification email").  
  3. פתח את המייל ולחץ על הקישור לאימות.  
- בלי אימות המייל אי אפשר לפרסם.

---

## 7. שמירת טיוטה

אחרי שמילאת את כל הנקודות — לחץ **"Save draft"** / **"שמור טיוטה"**.  
כשהכל מסומן ירוק (או בלי אזהרות חובה), אפשר ללחוץ **"Submit for review"**.

---

## סדר מומלץ

1. **חשבון** — הזן אימייל ליצירת קשר ואימות אותו.  
2. **נוהלי פרטיות** — מלא את כל ההצדקות וה־Single purpose מ־**PRIVACY_PRACTICES_STORE.md**, סמן אישור תאימות.  
3. **Store listing** — קטגוריה, תיאור מפורט (מלא).  
4. **Graphic assets** — אייקון 128×128, לפחות צילום מסך אחד.  
5. **שמור טיוטה** → **Submit for review**.

# בדיוק מה לעשות — צעד אחר צעד

אתה נמצא בדף **Privacy** (נוהלי פרטיות) בחנות Chrome. עשה **בדיוק** את זה, לפי הסדר.

---

## צעד 1: Single purpose description

1. מצא את השדה **"Single purpose description"**.
2. לחץ בתוך השדה.
3. **העתק** את כל השורות הבאות (מהשורה הראשונה עד האחרונה) **והדבק** בשדה:

```
This extension has a single purpose: to automatically save text that the user types in web forms and text areas, and to let the user restore that text with one click if the page was closed, the browser crashed, or the session expired. No data is sent to any server except optional license verification (license key only) for the paid upgrade. All saved text stays on the user's device.
```

---

## צעד 2: storage justification

1. גלול למטה עד **"storage justification"**.
2. לחץ בתוך השדה.
3. **העתק והדבק**:

```
The extension uses chrome.storage.local only to store: (1) the text snippets that the user has typed, so they can be restored later; (2) the user's settings (e.g. license key, checkout URL); and (3) the first-install date for the free trial. All of this data remains on the user's device. Nothing is sent to a server except an optional license verification request (license key and instance ID only) when the user activates the paid version. We do not use storage for tracking or advertising.
```

---

## צעד 3: activeTab justification

1. מצא **"activeTab justification"**.
2. לחץ בשדה.
3. **העתק והדבק**:

```
The extension needs "activeTab" to know which browser tab is currently active when the user clicks the extension icon. This is used only to: (1) display the list of saved text snippets for that tab's URL, and (2) send the restored text to that tab when the user clicks "Restore". We do not access tab content beyond inserting the restored text into the focused input or textarea.
```

---

## צעד 4: Host permission justification

1. מצא **"Host permission justification"**.
2. לחץ בשדה.
3. **העתק והדבק**:

```
The extension needs access to all URLs because it saves text that the user types in input fields and textareas on any website. The content script runs on every page only to: (1) listen for typing in input/textarea elements, (2) store that text locally on the user's device (chrome.storage.local), and (3) respond when the user chooses "Restore" by inserting text into the active field. We never collect or send the user's typed content to any server. Passwords and credit-card fields are explicitly excluded and never saved.
```

---

## צעד 5: Remote code

1. מצא את השאלה **"Are you using remote code?"**.
2. בחר את האפשרות: **"No, I am not using Remote code"** (לחיצה על העיגול / הרדיו).
3. אם מופיע שדה "Justification" מתחת — **תשאיר ריק**.

---

## צעד 6: What user data do you plan to collect?

1. גלול לקטע **"What user data do you plan to collect?"**.
2. יש רשימת סוגי נתונים (שם, אימייל, בריאות, תשלומים וכו').
3. **סמן רק** את האפשרות הזו (אם יש): **"Website content"** (תוכן אתר — טקסט, תמונות וכו').
4. **אל תסמן** שום דבר אחר (לא שם, לא אימייל, לא סיסמאות, לא מיקום, לא היסטוריה, לא פעילות).

---

## צעד 7: שלוש התיבות למטה (I certify that...)

1. גלול עד **"I certify that the following disclosures are true"**.
2. יש **3 תיבות סימון (צ'קבוקס)**.
3. **סמן את שלוש התיבות** (לחיצה בכל תיבה עד שיש V).

---

## צעד 8: Privacy policy URL

1. מצא את השדה **"Privacy policy URL"** (עם כוכבית — חובה).
2. אתה חייב put **קישור** לדף באינטרנט שמסביר את מדיניות הפרטיות.

**אם יש לך כבר אתר / דף נחיתה באינטרנט:**  
כתוב בשדה את הכתובת של דף הפרטיות, למשל:  
`https://האתר-שלך.com/privacy`  
או אם העלית את תיקיית landing ל-GitHub Pages:  
`https://שם-המשתמש.github.io/שם-הריפו/landing/privacy.html`

**אם עדיין אין לך שום אתר:**  
- יש קובץ **landing/privacy.html** בפרויקט — זה דף הפרטיות.  
- העלה את **כל תיקיית landing** (כולל privacy.html) ל-GitHub.  
- בהגדרות הריפו: Settings → Pages → Source: main branch → Save.  
- אחרי דקה־שתיים הקישור יהיה:  
  `https://שם-המשתמש.github.io/שם-הריפו/landing/privacy.html`  
  — **בדיוק את זה** תשים בשדה Privacy policy URL (החלף שם-המשתמש ו-שם-הריפו בשם האמיתי שלך).

**אם אתה לא רוצה להעלות ל-GitHub כרגע:**  
אפשר זמנית לשים קישור לדף פרטיות ציבורי כלשהו (למשל של שירות שמאפשר ליצור דף פרטיות חינם), ו**אחרי** שהתוסף מאושר — לעדכן לקישור האמיתי שלך (למשל landing/privacy.html כשתעלה אותו).

---

## צעד 9: שמירה

1. גלול למטה.
2. לחץ **"Save"** או **"שמור טיוטה"**.
3. אם יש עוד כרטיסיות עם אזהרות — תקן גם אותן.
4. כשהכל מסומן — לחץ **"Submit for review"**.
